Multi-Class Resume Classification using Classical Machine Learning and BERT

Yash Tyagi 2427030042 Manipal University Jaipur

"Can we automate the classification of resumes into 43 distinct professional categories using classical ML pipelines and modern transformer-based deep learning models?"

Problem Statement

Manual resume screening is one of the most time-consuming tasks in the recruitment pipeline. With thousands of applications per opening, HR professionals spend an average of 6–8 seconds per resume. This project addresses automated multi-class resume classification — assigning resumes to one of 43 professional categories — using both classical machine learning and deep learning (BERT) approaches to compare their effectiveness.

Literature Review

Prior works primarily used binary or few-class classification. The reference paper by Yadav et al. explored resume classification with TF-IDF + SVM on a smaller subset. Our work extends this to the full 43-class ResumeAtlas dataset and introduces BERT fine-tuning.

Research Gap & Innovation

Most existing systems handle fewer than 10 classes. Our approach benchmarks four distinct models (BoW+SVM, TF-IDF+SVM, GloVe+LR, BERT) across 43 categories, providing a comprehensive comparative analysis rarely found in literature.

Dataset Source

The ResumeAtlas dataset contains 13,389 resume samples categorized into 43 professional classes including Software Engineering, Data Science, Marketing, Finance, Healthcare, Education, and many more. Each resume is stored as raw text extracted from PDF/DOCX files. Preprocessing included lowercasing, removal of special characters, stopword elimination, and tokenization.

Classical ML Methods

BoW + SVM

Bag-of-Words feature extraction with Support Vector Machine (linear kernel) classifier. Captures word frequency patterns.

Text→ BoW→ SVM

TF-IDF + SVM

Term Frequency–Inverse Document Frequency weighting with SVM. Penalizes common terms, emphasizes discriminative words.

Text→ TF-IDF→ SVM

GloVe + Logistic Regression

Pre-trained GloVe word embeddings averaged per document, fed into a Logistic Regression classifier with multi-class support.

Text→ GloVe→ LR

Key Observations

  • Diagonal dominance — BERT's confusion matrix shows strong diagonal concentration, indicating high per-class accuracy across most of the 43 categories.
  • Common misclassifications — Semantically similar categories (e.g., "Data Science" vs "Machine Learning Engineer," "Sales" vs "Business Development") show moderate confusion across all models.
  • Rare class challenges — Categories with fewer training samples exhibit lower recall, particularly for GloVe+LR which struggles with underrepresented classes.
  • BERT advantage — BERT significantly reduces off-diagonal errors by leveraging contextual word meaning, distinguishing similar roles more effectively than BoW/TF-IDF.

Overfitting Analysis

Overfitting in Classical ML

  • BoW+SVM: 99.14% train → 79.50% test (gap: ~19.6%)
  • TF-IDF+SVM: 99.52% train → 82.26% test (gap: ~17.3%)
  • High-dimensional sparse features allow SVM to memorize training data patterns rather than learning generalizable decision boundaries.
  • With 43 classes and limited per-class samples, the model overfits to class-specific vocabulary.

BERT's Regularization

  • BERT: 95.20% train → 89.43% test (gap: ~5.8%)
  • Pre-trained representations provide strong inductive bias, reducing the need to memorize training examples.
  • Dropout layers (0.1) and attention mechanisms act as implicit regularizers.
  • Transfer learning from massive corpus enables better generalization to unseen resumes.

Conclusion

BERT significantly outperforms all classical machine learning approaches on the 43-class ResumeAtlas dataset, achieving 89.43% test accuracy vs. 82.26% for TF-IDF+SVM. Contextual embeddings capture semantic nuances that sparse bag-of-words representations cannot, making transformer-based models the superior choice for multi-class resume classification.
